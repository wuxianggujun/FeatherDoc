1. 目标是检查 `{{DOC_NAME}}` 的最终肉眼可见效果，而不是只看 XML、结构树、单元测试或程序返回值。

执行顺序：

1. 验证 `{{DOCX_PATH}}` 存在，并在输出中明确写出绝对路径。
2. 启动 Word，并显式打开该文档。
3. 验证：
   - Word 窗口存在；
   - 焦点窗口是 Word；
   - 当前活动文档就是 `{{DOCX_PATH}}`；
   - 当前界面不是开始页、空白文档页或最近文件页。
4. 调整到适合排版审查的视图和缩放比例，确保屏幕能看到真实页面主体。
5. 对每一页或关键页面执行固定动作：
   - 重新验证焦点仍是 Word；
   - 重新验证当前文档仍然是 `{{DOCX_PATH}}`；

6. 重点检查：
   - 一级标题、二级标题位置是否正确；
   - 正文是否有异常空行、错位、截断、重叠、丢字；
   - 表格边框、列宽、行高、跨页、分页断裂是否异常；
   - 图片、图注、文字环绕是否异常；
   - 页码、页眉、页脚是否正常；
   - 是否有元素重叠、超出页面、分页突变、大片空白。
