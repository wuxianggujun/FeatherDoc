Getting Started
===============

Install And Build
-----------------

.. code-block:: sh

   cmake -S . -B build
   cmake --build build
   cmake --install build --prefix install

Top-level builds enable ``BUILD_CLI`` by default, so ``featherdoc_cli`` is built
unless ``-DBUILD_CLI=OFF`` is passed explicitly.

Minimal C++ Usage
-----------------

.. code-block:: cpp

   #include <featherdoc.hpp>

   int main() {
       featherdoc::Document doc{"input.docx"};
       if (doc.open()) {
           return 1;
       }

       doc.paragraphs().add_run("Generated by FeatherDoc");
       return doc.save_as("output.docx") ? 1 : 0;
   }

Common Entry Points
-------------------

* Use :doc:`api/document` for open/save, sections, headers, footers, inspection,
  styles, numbering, and document-wide features.
* Use :doc:`api/content_blocks` for paragraphs and runs.
* Use :doc:`api/tables` for table, row, and cell editing.
* Use :doc:`api/templates` for bookmarks, content controls, template slots, and
  schema validation.
* Use :doc:`api/fields_links_reviews` for fields, hyperlinks, comments,
  footnotes, endnotes, and revisions.
* Use :doc:`api/images_sections` for inline images, floating images, and section
  page setup.

Installed Package Notes
-----------------------

The installed package carries project-facing metadata, visual-validation preview
assets, repro guides, release-artifact templates, and legal files under
``share/FeatherDoc``. When browsing an installed tree, start with:

.. code-block:: text

   share/FeatherDoc/VISUAL_VALIDATION_QUICKSTART.zh-CN.md
